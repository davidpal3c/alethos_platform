# TASK-0002 — Storage Tier Mount Configuration (Phase 0B)

**Status:** Ready

## Objective

Implement the **remaining three storage tiers** (platform NVMe, database NVMe, backup HDD): partitions, ext4, UUID-based `/etc/fstab`, directory layout, and bind mounts for k3s/observability paths. Boot tier (RAID1) is already live — **must not be reformatted**.

---

## Live node truth (alethos-node-01)

Per `context/project-context.json` after TASK-0001:

| Role | Devices (confirm with `lsblk`) |
|------|--------------------------------|
| Boot RAID1 | **sda** (Intel Pro 1500) + **sdc** (Intel 520) → md0/md1/md2 |
| Backup HDD (1TB) | **sdb** — **not sdc** |
| Platform | 240GB WD SN530 NVMe (typically `nvme0n1` or similar) |
| Database | 1TB NVMe (typically `nvme1n1` or similar) |

**Critical:** Earlier docs assumed backup on `sdc1`. On this node **sdc is the second RAID SSD**. Backup tier partition must be on the **1TB SATA HDD (sdb)**. Always verify by model and size before `mkfs`.

---

## Mount architecture

| Tier | Mount | Partition (after create) |
|------|-------|---------------------------|
| Platform | `/platform` | Whole-disk or single primary on 240GB NVMe |
| Database | `/data` | Single primary on 1TB NVMe |
| Backup | `/backups` | Single partition on **sdb** (HDD) |

Filesystem: **ext4** for these tiers (aligns with pending FS strategy; ZFS deferred).

All persistent entries in `/etc/fstab` must use **UUID=** (from `blkid`).

---

## Directories

**Under `/platform`:**  
`rancher`, `kubelet`, `containerd`, `prometheus`, `loki`

**Bind mounts (after base mount):**

| Source | Target |
|--------|--------|
| `/platform/rancher` | `/var/lib/rancher` |
| `/platform/kubelet` | `/var/lib/kubelet` |
| `/platform/containerd` | `/var/lib/containerd` |
| `/platform/prometheus` | `/var/lib/prometheus` |
| `/platform/loki` | `/var/lib/loki` |

**Under `/data`:**  
`postgres`, `postgres_wal`, `redis`

**Backup:** `/backups` — dumps, archives; no hot DB/platform data.

---

## Constraints

• Do not place platform high-write paths on boot RAID.  
• Do not use RAID members (sda/sdc beyond existing layout) for ad-hoc partitions.  
• Base mount must appear in `fstab` **before** bind-mount lines for that tree.

---

## Acceptance criteria

1. Reboot persists `/platform`, `/data`, `/backups` and all bind mounts.  
2. UUIDs only in `fstab` for these filesystems.  
3. Ownership/mode suitable for later k3s (document any `chown` chosen).  
4. Build report + doc updates listed in `tasks/todo.md`.
